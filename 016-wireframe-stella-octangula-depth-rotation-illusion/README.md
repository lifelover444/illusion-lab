# 星形八面体纵深歧义旋转错觉

这是 Illusion Lab 的独立实验 016：由两个方向相反、互相穿插、完全同材质的正四面体组成一枚稀疏的星形八面体粒子晶体。几何从立方体八角点构造，并把原始体对角线 `(1,1,1)` 对准世界 Z 轴；整体只有 8 个外部顶点、12 条真实四面体棱和 8 个用于表面粒子采样的大等边三角面。

两组棱在六个变换后的立方体面心真实相交。交点不新增顶点、不拆边、不做断线、桥接或前后遮挡，也不用颜色区分四面体。正交相机、全部可见的背棱、无阴影、无透视缩放和单一刚体匀速旋转，让两条交叉线的纵深解释保持可交换。

画面改用实验 011 的粒子质感：不直接绘制任何曲线，12 条棱由 960 个不透明微型八面体粒子断续拼成，并以少量同样离散的粒子 halo 维持低清可读性。八个大三角面由 6400 个不透明微粒按面积均匀采样，其中 4160 个是较小、较暗的彩色底尘，2240 个是放大约 1.7 倍的亮彩晶粒。所有棱、halo 与表面粒子共享一个固定在物体坐标中的 24 色阶循环虹彩场；同一三维位置始终拥有同一色相，渐变不会按四面体、面、相机纵深或时间变化。整个彩虹场随单一刚体一起旋转，因此呈现连续虹彩珠串，却不在交叉点提供前后编码。

场景中没有透明面板，也没有完整的非凸实体外壳，因此穿插面不会因 alpha 重叠而变暗，也不会读成空心玻璃。场景仍没有灯光、阴影、玻璃折射、Fresnel、反射、地面、景深、内部尘、顶点亮星或独立粒子运动。交叉处不拆线、不桥接，所有粒子均锁定在同一个刚体 rig 下。

## 结构

```text
src/stellaOctangulaGeometry.ts          # compound 几何、坐标基、交点、投影与循环数学
tests/stellaOctangulaGeometry.test.ts   # 拓扑、度量、对称、交点、投影与低清范围测试
scripts/create_scene.py                 # 独立 Blender 场景、静帧、接触表与循环检查
scripts/stella-octangula-config.json    # 唯一的 360×640 低清 profile
scripts/run-blender.mjs                 # Blender 入口与低清 MP4 规格验证
scene/                                  # 生成的 .blend
output/                                 # 低清静帧、检查数据、接触表和 MP4
```

## 命令

在仓库根目录运行：

```powershell
npm run test:016
npm run build:016
npm run scene:016
npm run render:016:stills
npm run render:016:preview
```

默认 Blender 路径为 `D:\software\Blender 4.5 LTS\blender.exe`，也可设置 `BLENDER_EXE`。本实验有意只提供 360×640、15 fps、24 秒、10 samples 的低清 preview；不存在 final、HD 或 720×1280 profile/命令。

正交范围最终使用 5.87：六个非轴顶点在完整旋转中形成最大宽度约 3.266 的正六边形，而 9:16 画面在范围 5.3 时横向容量仅约 2.981。为保留全部外尖和完整棱，构图选择不裁切、不压缩正四面体几何，代价是最大竖向占比约 58.6%。

视频包含两整圈。编码的第 360 帧停在闭合姿态前一个 2° 角步，未编码的下一帧才与首帧严格一致，从而避免重复尾帧停顿。

动画从 0° 三重对称姿态开始，使首帧先清楚读成经典星形八面体，再进入纵深重排；原先容易形成倾斜四边形的 30° 姿态仅作为旋转过程中的普通相位出现。
