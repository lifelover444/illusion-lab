# 012 无缝三角双锥点云深度旋转错觉设计

## 目标

在 `012-seamless-triangular-bipyramid-depth-rotation-illusion/` 中创建一个自包含 Blender 实验：一颗由细小冷白微光点构成的完整三角双锥，在纯黑背景中绕上下尖端连成的竖直轴匀速旋转。正交投影、均匀材质和无阴影环境应削弱可靠的前后线索，让观众在看清旋转的同时有机会感知顺时针与逆时针方向翻转。

本实验只制作三角双锥，不制作正八面体或对照版本。它必须与 004 的沙漏结构明显不同：012 的两个三棱锥共用中部底面并形成一个完整双尖外壳，不允许上下分离、开合、呼吸或相对运动。

## 几何

几何体恰好包含五个唯一顶点和六个唯一外露三角面：

- 顶点 `(0, 0, +H)` 与底点 `(0, 0, -H)` 关于 `z = 0` 对称。
- 三个中部顶点位于 `z = 0`，外接圆半径 `R = 1.0`，从 `initialPhase = 30°` 开始每隔 `120°` 排列。
- 首版 `H = sqrt(2) * R`，使六个外露面接近等边三角形。
- 上部三个面由顶点与中部三角形的三条边连接；下部三个面由底点连接同样三条边。
- 内部中部三角底面不存在，几何拓扑按一个统一外壳定义。
- 所有三角面顶点顺序统一校正为法向朝外。

实现提供纯 TypeScript 几何函数，作为顶点、面、面积、法向、采样、投影和循环逻辑的可测试权威定义。Blender 脚本使用相同公式和配置值复现几何。

## 点云外壳

采用已确认的方案 A：六个外露三角面通过同一个采样流程生成，并把全部微型点几何合并为一个点云网格对象。上下两半不是独立成品对象，所有点共用一个冷白发光材质，并由同一个旋转 Rig 驱动。

- 总点数使用 `5700`，每面 `950` 点。
- 每个面使用分层低差异重心采样，并加入受限的微小确定性扰动；所有采样由固定种子控制。
- 三角采样采用面积均匀映射，不直接把两个未经折叠的均匀随机数当作重心坐标。
- 点的位置和大小在场景生成后固定，不闪烁、不重采样、不流动。
- 点尺寸在 `0.0045–0.009` 范围内轻微变化，分布偏向较小值。
- 单个点以微型八面体表示，所有点合并到同一 mesh；不使用顶点大光球。
- 不生成任何专用边点、连续棱线、中部腰线、内部粒子或额外装饰几何。
- 公共中部边只由相邻表面的面内采样自然接近，不单独采样，因此不会形成重复边线或密度翻倍。

## 材质与渲染

所有点使用一个冷白 Emission 材质：`pointColor = [0.82, 0.91, 1.0]`，首版 `emissionStrength = 1.22`。世界背景为纯黑，无灯光、阴影、环境遮蔽、反射、地面、倒影、颜色渐变、深度亮度变化或面朝向亮度变化。

Blender Eevee Next 输出经过非常轻的 Fog Glow 合成，仅柔化单点边缘，不把相邻点连成白面或在中部形成发光腰带。颜色管理保持所有面一致，不引入正背面提示。

## 镜头与构图

使用固定正交相机，正面看向原点，`cameraTargetZ = 0`，首版 `cameraElevationDegrees = 0`。不使用透视、镜头移动、变焦、景深或相机旋转。

输出为 `9:16` 竖屏。正交范围按 `H = sqrt(2)` 和安全留白设置，使主体高度约占画面 `64%–66%`，水平宽度在旋转相位中保持可读，上下尖端远离边缘，主体中心位于画面中心。

## 动画与循环

唯一动画是完整点云外壳绕世界 `Z` 轴匀速旋转：

- 每圈 `5.4` 秒，总时长 `10.8` 秒，恰好两圈。
- 预览为 `360 × 640`、`15 fps`、H.264 MP4，共 `162` 个输出帧。
- 最终配置为 `720 × 1280`、`30 fps`、H.264 MP4，共 `324` 个输出帧，但本轮不渲染。
- 动画在首帧插入 `0°`，在 `loopFrame = frameEnd + 1` 插入 `720°`，关键帧使用线性插值。
- 输出的最后一帧位于循环端点之前；循环后的下一帧与首帧旋转角完全一致。

不添加开合、缩放、顶点移动、波动、上下反向旋转、漂浮、相机动画、粒子动画、颜色或亮度脉冲。

## 静帧、接触表与预览门槛

第一轮生成：

1. Blender 场景文件。
2. `0°、30°、60°、90°、120°、180°、240°、300°、360°` 九个关键相位 PNG。
3. 将九个相位按角度顺序横向排列的接触表 PNG。
4. 静帧检查通过后生成低清完整预览 MP4。

静帧检查确认：`0°` 与 `360°` 像素一致；任何相位都不是退化平面三角形；上下尖端始终清楚；中部三个顶点轮流提供运动参照；不存在发光腰线、实体白面或可靠的最前面分面提示；轮廓一眼可读为完整双尖水晶。

若中部出现亮腰线，先降低边界附近重复视觉密度或 Bloom，不增加装饰。若形体过细，只把 `H` 调整至 `1.25–1.35`。若形体过普通，只调整点尺寸、点密度和构图。

## 测试策略

Vitest 采用测试优先实现，至少覆盖：

1. 恰好五个唯一顶点。
2. 恰好六个唯一三角面。
3. 上下顶点关于 `z = 0` 对称。
4. 三个中部顶点全部位于 `z = 0`。
5. 中部角度间隔为 `120°`。
6. 六面面积在浮点误差内相同。
7. 所有三角面法向朝外。
8. 采样点位于对应三角形内部。
9. 几何 API 不生成公共中部边的专用边线。
10. 动画首帧与循环后下一帧的归一化旋转角一致。
11. 九个固定相位的正交投影宽度均高于安全阈值。
12. 固定种子生成的点位置与大小完全可复现。

构建通过 `tsc` 验证。Blender 阶段还检查 `.blend` 文件、九张静帧、接触表、预览 MP4 的存在和媒体规格，并通过图像比较验证 `0°` 与 `360°` 一致。

## 明确不做

本轮不渲染最终版，不制作正八面体、对比画面、实体透明材质、连续棱边、中部三角腰线、彩色描边、星空、文字、UI、地面阴影或任何额外几何结构。最终视频必须等待用户检查预览 MP4 和接触表后再执行。
