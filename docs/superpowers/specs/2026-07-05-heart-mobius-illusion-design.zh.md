# 心形莫比乌斯错觉设计

日期：2026-07-05
项目：`005-heart-blender-illusion`
状态：设计已确认，尚未实现

## 概要

本实验会在 Blender 中制作一个发光的心形莫比乌斯环。主体不是剪影，也不是实心水晶心，而是一条沿心形路径闭合的半透明连续丝带，并在闭环路径上完成 180 度莫比乌斯扭转。

目标是在绕竖直轴旋转时，同时保留连续空间运动和深度暧昧感。由于丝带没有明确的前后标记、使用正交相机，并且避开单侧光照线索，观看者应能把同一段线性旋转感知为顺时针或逆时针。

## 目标

- 在 `005-heart-blender-illusion` 内通过代码程序化生成模型。
- 第一版实现时，从正面看必须能清楚识别为心形。
- 使用连续丝带表面，让运动看起来是真正旋转，而不是帧间形变。
- 弱化前后、近远、表面正反等线索，保留双稳态方向感知。
- 通过现有 workspace 命令渲染预览版和最终竖屏视频。

## 非目标

- 不做纯黑剪影。
- 第一版不做物理真实的玻璃或水晶心。
- 不使用普通立方体、实心块体或常规不透明多面体作为主体。
- 不加入强投影、透视深度、非对称纹理或单侧高光。
- 不通过手工关键帧形变来假装旋转。

## 用户可见效果

最终渲染应展示一个粉色或洋红色的发光心形丝带，漂浮在深色背景中。12 秒视频内，整体绕竖直轴匀速旋转两圈。

正面阶段，心形轮廓应清晰可读。侧面阶段，形体应被压缩，但仍能看到连续扭转的带面，而不是塌成一条平线。背面阶段，心形应以相近视觉强度再次出现。当观看者无法稳定判断它到底在顺时针还是逆时针旋转时，错觉成立。

## 几何结构

模型从参数化心形中心线生成，初始采样数约为 240 个点。二维心形曲线放在 `X/Z` 平面中，`Z` 为竖直方向，丝带的深度坐标使用 `Y`。

每个中心线采样点需要：

- 根据相邻曲线点计算切线。
- 在心形平面内计算局部法线。
- 用局部法线叠加莫比乌斯扭转角，生成丝带横向方向。
- 让扭转角在完整闭环上累计推进 180 度。
- 每个采样点生成两个丝带边缘顶点。
- 用四边面连接相邻采样点。
- 将最后一个采样点闭合回第一个采样点。

第一版使用一个 180 度扭转。实现中可以暴露 `mobiusTwistAmount` 用于调参，范围大约为 150 到 210 度，但默认仍为 180 度。

丝带应足够薄，以保持精致和暧昧；也要足够宽，避免只像一条线。初始带宽目标约为心形高度的 10% 到 16%。

## 材质

材质应主要依赖发光和透明度，而不是真实方向光照。

默认视觉：

- 半透明粉色丝带面。
- 更亮的洋红色或柔白色边缘线。
- 正反面视觉处理一致。
- 非常轻微的 bloom 或辉光。
- 深色背景。

第一版明确排除会锁定方向的材质选择：

- 不使用玻璃折射。
- 不使用金属式高光。
- 不使用固定在单侧的强亮点。
- 不让正反面使用不同颜色。
- 不使用会标记环绕方向的纹理图案。

## 相机与运动

相机使用正交投影，从负 `Y` 方向看向物体。模型绕竖直 `Z` 轴旋转。

默认运动参数：

- `rotationCycleSeconds`：6
- `totalSeconds`：12
- `easing`：linear
- `rotationAxis`：竖直 `Z`
- `cycles`：2

正交相机缩放应让心形在 9:16 竖屏画面中留有舒适边距。相机不能使用透视投影，因为近大远小会暴露深度顺序。

## 场景

第一版应避免物理地面。如果后续需要空间参照，只能使用非常淡、与旋转相位关联的辉光或雾化背景，而不是固定投影。

照明应保持极简，因为主体主要依赖自发光。任何环境光都应对称且柔和。

## 组件

实现应保持实验目录自包含：

- `scripts/heart-illusion-config.json` 存放渲染、模型、材质、相机和运动参数。
- `scripts/create_scene.py` 根据配置构建 Blender 场景。
- `scripts/run-blender.mjs` 调用 Blender 并处理场景生成和渲染配置。
- `src/heartIllusionPlan.ts` 暴露可测试的规划和配置预期。
- `tests/heartIllusionPlan.test.ts` 验证保护错觉效果的约束。

## 数据流

1. Node runner 接收 `scene`、`preview` 或 `final`。
2. 它加载 JSON 配置，并选择对应渲染 profile。
3. 它启动 Blender，并传入 `create_scene.py`、运行模式和配置路径。
4. Blender 脚本创建莫比乌斯心形网格、材质、相机、动画和渲染设置。
5. 预览版和最终版模式渲染到配置中的输出路径。

## 错误处理

如果找不到 Blender，runner 应给出清晰错误信息。Blender 脚本应在生成场景前验证必要配置字段。无效值，例如非正采样数、非正带宽、不支持的相机投影或非线性缓动，应尽早失败。

需要输出目录时应自动创建。显式执行渲染命令时，可以覆盖已有渲染输出。

## 测试

单元测试应在不依赖 Blender 的情况下覆盖设计约束：

- 实现开始后，项目状态从 planning-only 切换为 modeling-started。
- 主体是莫比乌斯心形环，不是剪影或实心水晶心。
- 相机保持正交投影。
- 运动保持绕竖直轴线性旋转。
- 默认渲染时长刚好包含两次完整旋转。
- 拒绝透视相机、投影、非对称纹理、玻璃折射和单侧高光等锁定方向的线索。
- 保留自发光材质、透明表面、正反一致处理和连续可见丝带等维持暧昧感的线索。
- 配置值应保证侧面阶段的丝带表面仍然可读。

视觉验收应检查 0、90、180 和 270 度附近的画面：

- 0 度：清晰正面心形。
- 90 度：压缩但连续的扭转丝带。
- 180 度：心形以相近强度重新出现。
- 270 度：第二个压缩侧面阶段，仍保持方向暧昧。

## 验收标准

- `npm run test:005` 通过。
- `npm run build:005` 通过。
- 实现开始后，`npm run scene:005` 能创建 Blender 场景。
- `npm run render:005:preview` 能创建短预览视频。
- 渲染主体是发光莫比乌斯心形环，不是剪影、立方体或实心水晶心。
- 渲染避免强透视、投影、方向性高光和非对称标记。
- 旋转具有连续感，并且可以被合理解释为两个方向中的任意一个。
