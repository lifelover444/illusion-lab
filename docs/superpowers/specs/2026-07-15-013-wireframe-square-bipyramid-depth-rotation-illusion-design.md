# 013 冰晶星尘四角双锥深度旋转错觉设计

## 目标

新建独立实验 `013-wireframe-square-bipyramid-depth-rotation-illusion/`，使用与 008 相同的 Blender 脚本化场景生成、竖屏循环渲染和 TypeScript 几何测试技术。成品是一颗上下对称的四角双锥，视觉质感贴近用户提供的参考图：黑色背景、深色透明外壳、冰晶虹彩发光棱线和密集星尘颗粒。模型绕竖轴连续旋转时，不提供确定前后关系的视觉线索，使观看者可以将同一运动感知为顺时针或逆时针。

## 形体

- 模型是一个完整四角双锥，不是两个独立对象的拼接动画。
- 四个腰部顶点位于同一水平面并组成正方形。
- 上下尖端位于同一竖轴，距腰部平面的距离相等。
- 八个三角面共同构成闭合且上下对称的外壳。
- 中间正方形腰线是明确的结构线，亮度和可读性接近参考图。
- 上下两半使用相同材质、线条和粒子语言，不出现拼接缝或材质差异。

## 视觉设计

### 背景与构图

- 使用纯黑背景，不设置地面、环境物体、文字或方向提示。
- 使用竖屏居中构图和正交相机。
- 相机保持轻微俯视角，使正方形腰部保持可读，同时避免强透视缩短。
- 主体在旋转全过程中留有稳定的安全边距，上下尖端不接触画面边缘。

### 透明外壳

- 八个三角面使用极淡的深蓝黑透明材质，提供参考图中的薄玻璃/烟晶质感。
- 透明面不产生投影、明显折射位移、强 Fresnel 高光或稳定的前后明暗差。
- 前后面都可见，透明度保持足够低，避免面遮挡锁定旋转方向。
- 外壳只承担整体感和轻微色层，不承担主要读形；读形由腰线、斜棱与星尘共同完成。

### 发光棱线

- 描绘四条腰线、四条上锥斜棱和四条下锥斜棱。
- 使用冷白细亮核心，辅以非常克制的青、蓝、紫和粉色虹彩变化。
- 所有前后棱线近似等亮，不按深度排序降低背边亮度。
- Bloom 只产生短距离柔光，不形成大面积霓虹光带。
- 腰部正方形保持清晰发亮，但线宽不超过斜棱到足以成为唯一的方向追踪锚点。

### 星尘颗粒

- 大量微小颗粒均匀附着在八个三角面上，随模型作为刚体一起旋转。
- 颗粒以冷白为主，少量使用青、蓝、紫和粉色，颜色分布无方向性。
- 粒径、亮度和位置带有固定随机变化，但不设置沿旋转方向传播的闪烁动画。
- 腰线和棱线附近可以有轻微密度提升，以接近参考图的冰晶边缘感。
- 前后表面颗粒保持近似等亮，避免遮挡和明暗排序暴露真实方向。

## 双稳态旋转设计

- 整个双锥作为一个刚体绕世界竖轴匀速旋转，相机固定。
- 使用正交投影，避免透视尺度变化提供确定深度线索。
- 不使用地面阴影、固定反射亮斑、方向箭头、前后色差或单侧装饰。
- 棱线和粒子均允许背侧可见，并保持前后视觉权重接近。
- 透明外壳不应比线框和点云更强；若静帧或预览出现唯一稳定方向，应优先降低面透明度和高光，再考虑降低腰线光晕。
- 旋转周期为 12 秒；24 秒成片包含两整圈。循环后的下一帧与首帧姿态一致。

## 技术架构

项目延续 008 的独立工作区结构：

```text
013-wireframe-square-bipyramid-depth-rotation-illusion/
├─ scripts/
│  ├─ create_scene.py
│  ├─ square-bipyramid-config.json
│  └─ run-blender.mjs
├─ src/
│  └─ squareBipyramidGeometry.ts
├─ tests/
│  └─ squareBipyramidGeometry.test.ts
├─ scene/
├─ output/
├─ package.json
├─ tsconfig.json
├─ vitest.config.ts
└─ README.md
```

### 场景生成

`create_scene.py` 读取 JSON 配置，生成唯一双锥几何骨架、透明三角面、发光棱线、表面星尘、正交相机、合成 Bloom 和刚体旋转动画。随机粒子使用固定种子，保证测试、静帧和视频结果可复现。

### 配置

`square-bipyramid-config.json` 集中保存：

- 预览和最终输出规格；
- 腰部半径与上下锥高度；
- 棱线核心半径、光晕半径与虹彩比例；
- 面透明度与深蓝色调；
- 粒子数量、粒径、抖动、边缘密度提升与随机种子；
- 正交相机比例、方位角、俯仰角和目标点；
- 旋转周期。

### 命令入口与错误处理

`run-blender.mjs` 提供 `scene`、`stills`、`preview` 和 `final` 四个入口。它在 Blender 可执行文件不存在、命令未知或 Blender 返回非零状态时输出明确错误并返回失败状态。默认 Blender 路径与 008 相同，也允许通过 `BLENDER_EXE` 覆盖。

仓库根目录增加 013 workspace 及以下命令：

```text
build:013
test:013
scene:013
render:013:stills
render:013:preview
render:013:final
```

## 输出规格

- 静帧：覆盖正面、斜侧、边缘压缩和完整循环端点的关键角度，并生成接触表。
- 预览：`360 × 640`、15 fps、24 秒、两整圈。
- 最终版：`720 × 1280`、30 fps、24 秒、两整圈。
- 场景文件和渲染结果保存在项目内部的 `scene/` 与 `output/`，不提交生成物。
- 实现阶段先生成场景、静帧接触表和低清预览；未经视觉检查不执行最终渲染。

## 测试与验收

### 自动测试

- 顶点数量为 6，三角面数量为 8，结构索引有效。
- 四个腰部顶点共面并组成正方形。
- 上下尖端关于腰部平面对称，并与正方形中心共轴。
- 每个腰部顶点同时连接上尖端、下尖端和两个相邻腰部顶点。
- 正交投影下，成对对称顶点满足预期镜像关系。
- 旋转角在一周期后闭合；24 秒输出恰好包含两整圈。

### 视觉验收

- 静帧中主体始终完整、居中且无裁切。
- 上下两半清楚读作同一个四角双锥。
- 中间正方形腰线清晰发亮，符合参考图的结构强度。
- 棱线呈细亮冰晶质感，虹彩克制，不变成粗霓虹管。
- 八个面能感知到极淡深色透明层，但不会形成明确前后遮挡。
- 星尘密集且细碎，前后表面视觉权重接近。
- 预览中旋转连续、匀速、首尾无跳变。
- 在没有文字提示的情况下，同一段旋转具备顺时针和逆时针两种合理知觉。

## 范围边界

- 不制作网页控制器、实时交互、音频或额外场景元素。
- 不使用真实物理折射、强反射灯箱、地面阴影或透视相机。
- 不复制参考图本身作为贴图；参考图只用于形体下半部分和视觉质感指导。
- 不改造其他编号实验的内部实现；只在根 workspace、脚本命令、README 和忽略规则中添加 013 所需入口。
